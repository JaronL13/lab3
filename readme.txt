0) Hva er fordelen med et eksisterende JavaScript bibliotek som jQuery i motsetning til å
skrive en “rå” JavaScript for dynamisk endring av HTML sider?
SVAR: Man slipper å bruke så lange kommandoer som gjør det mer programmerings vennlig samtidig som det åpner opp for flere muligheter.

1) Les denne artikkelen
http://www.ibm.com/developerworks/web/library/wa­htmlmark/index.html og svar på følgende
spørsmål:

Hva er sterke sider til HTML/CSS og hva er sterke sider til Canvas? I hvilke
situasjoner er det meningsfult å bruke Canvas?
SVAR: 	HTML/CSS sterke sider = Noe enklere og simplere muligheter innen forandring av tekst fonts og diverse, 
				Blir plukket opp av søkemotorer, kan observere hyperlinker og alt av webelementer og designet for programmer.
				Raskere å sette sammen en prototype.
				
				
	Canvas sterke sider = 	Enklere å få mer grafiske effekter på tekst og design. 
				Lettere å bruke over flere forskjellige enheter og raskere til å vise sider når hastighet er av høy viktighet.
				

Canvas er altså mer meningsfult å bruke når man skal lage gode grafiske elmenter som kjører bedre og lettere,
og kan også kombineres med HTML hvis det gjør det praktisk lettere.

2) Hvilken grafisk presentasjonsmåte (se Innledning til denne oppgaven) hadde du sagt at
HTML5 Canvas representerer? Forklar.
SVAR: En god dyp og gjennomført presenteasjonsmåte som går helt ned til pixelnivå og som er veldig variabel til grafisk bruk
